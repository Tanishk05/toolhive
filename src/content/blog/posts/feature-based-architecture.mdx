## Build around boundaries

Feature-based boundaries make it easier to scale because each domain owns its data, UI, and server logic.

## A small example

```ts
// A feature module should expose an intentional API surface.
export const blog = {
  getPosts,
  getCategories,
  getRelatedPosts,
};
```

## What to avoid

- shared dumping grounds
- leaky cross-domain imports
- page-specific data logic

## What to prefer

- explicit feature APIs
- server-only data loaders
- stable content contracts
